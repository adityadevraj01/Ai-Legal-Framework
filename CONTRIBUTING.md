## üìù Contribution Guidelines

### Opening Issues

When opening an issue, please:

1. **Use a clear, descriptive title**
   - Good: "Clarify definition of 'significantly modified' in Section 4"
   - Poor: "Question about definitions"

2. **Provide context**
   - Which section are you referencing?
   - What problem does your suggestion solve?
   - What alternative approaches have you considered?

3. **Label appropriately**
   - `enhancement` - Improvements to existing provisions
   - `bug` - Errors, inconsistencies, or ambiguities
   - `question` - Need clarification
   - `legal` - Legal interpretation or compliance issues
   - `technical` - Technical implementation concerns
   - `discussion` - Broader policy discussions

### Submitting Pull Requests

For substantial changes:

1. **Open an issue first** to discuss the proposed change
2. **Fork the repository** and create a new branch
3. **Make your changes** with clear commit messages
4. **Update relevant sections** including table of contents if needed
5. **Test your changes** (ensure markdown renders correctly)
6. **Submit the PR** with a description linking to the original issue

For minor fixes (typos, formatting):
- Direct PRs are welcome without prior discussion

### Writing Style

When proposing changes, please follow these guidelines:

**Clarity:**
- Use plain language accessible to non-experts
- Define technical or legal terms
- Avoid unnecessary jargon

**Structure:**
- Maintain logical flow between sections
- Use consistent formatting
- Number provisions clearly

**Tone:**
- Professional and authoritative
- Neutral and objective
- Solution-oriented

**Precision:**
- Be specific about requirements
- Avoid vague language like "should consider"
- Clearly distinguish obligations from recommendations
